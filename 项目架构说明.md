# TechLex 项目架构说明

## 1. 用户如何使用

```text
打开网页
→ 粘贴英文技术文档
→ 提取专业术语
→ 为术语生成定义、例句和应用场景
→ 保存到浏览器卡片库
→ 进入复习模式
→ 根据答对或答错调整复习权重
```

MVP 使用“粘贴文本”而不是上传文件，避免第一版同时处理 PDF、Word、编码和文件大小等问题。文件和 URL 导入放入 Roadmap。

## 2. 为什么仓库不只有代码

仓库包含三类内容：

1. **应用代码**：React 页面、服务端接口、本地存储和复习逻辑。
2. **项目文档**：解释产品范围、设计决策、开发步骤和后续计划。
3. **Prompt 资产**：记录术语提取和卡片生成 Prompt 的目标、契约与迭代。

Prompt 文档在 `prompts/`，实际运行模板在 `server/promptTemplates.js`。两处必须保持同步。

## 3. 技术方案

- 前端：React + Vite + Tailwind CSS
- AI：DeepSeek JSON Output API
- 服务端：Vercel Functions
- 数据：浏览器 localStorage
- 测试：Vitest + React Testing Library

前端不能保存 API Key。React 只调用自己的 `/api` 接口，Vercel Function 从服务端环境变量读取 `DEEPSEEK_API_KEY`。

## 4. 分层

| 层 | 位置 | 职责 |
|---|---|---|
| 页面 | `src/components/` | 输入、卡片展示和复习交互 |
| 协调 | `src/App.jsx` | 管理卡片、模式、加载和错误状态 |
| 前端服务 | `src/services/` | 请求 `/api` 并校验业务结果 |
| 数据 | `src/storage/` | localStorage CRUD |
| 业务逻辑 | `src/logic/` | 答案标准化、权重和抽题 |
| 服务端接口 | `api/` | 校验 HTTP 输入并输出统一 JSON |
| AI 适配 | `server/` | Prompt、DeepSeek 调用、超时和错误映射 |

## 6. 开发模式

### 模拟模式

```bash
npm run dev
```

该命令固定使用模拟 AI 数据，不消耗 API，也不依赖服务端函数。

### 完整模式

```bash
npm run dev:full
```

该命令使用 Vercel 本地开发环境和真实 DeepSeek API。真实密钥只放在 `.env` 或 Vercel 项目环境变量中。

## 7. Card 数据

卡片包含术语、定义、例句、使用场景、原文句子、创建时间、复习权重和答题统计。数据保存在 `techlex.cards.v1`。

新卡片权重为 `1`；答错增加权重，答对降低权重；抽题按权重随机。这是第一版启发式规则，不是完整 SM-2 算法。

## 8. 当前边界

- 不做账号、数据库和云同步。
- 不做文件或 URL 导入。
- 不做多模型切换。
- 不做完整间隔重复日期调度。
- 公开部署后仍需要根据使用量增加限流和费用保护。
