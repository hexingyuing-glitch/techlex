# Roadmap

Roadmap 按可验证结果排列，而不是按功能数量排列。只有代码、基本异常处理和人工验证全部完成后，任务才标记为已完成。

## 当前状态

- [x] 明确产品定位和一周 MVP 边界
- [x] 建立仓库目录、设计文档、开发日志和 Prompt 资产结构
- [ ] 完成可运行的 React + Vite 基础页面
- [ ] 完成“输入文本 → 生成卡片 → 本地保存 → 复习”的端到端闭环

## v1：一周 MVP

### Day 1：初始化与基础 UI

- [ ] 配置 React、Vite 和 Tailwind CSS
- [ ] 完成输入区、操作按钮、卡片区和页面布局
- [ ] 加入输入为空、处理中和失败状态

### Day 2：术语提取

- [ ] 封装 Claude API 调用
- [ ] 实现术语提取 Prompt 和 JSON 校验
- [ ] 展示提取结果并处理超时、限流和格式错误

### Day 3：卡片生成

- [ ] 为术语生成定义、例句和应用场景
- [ ] 完成卡片列表与卡片组件
- [ ] 支持单项失败重试，避免整批结果丢失

### Day 4：本地卡片库

- [ ] 实现 localStorage 的读取、新增、更新和删除
- [ ] 对术语做基础去重
- [ ] 处理损坏数据和存储不可用情况

### Day 5：复习模式

- [ ] 支持“看定义猜术语”
- [ ] 提交答案后显示正确结果和反馈

### Day 6：简单间隔重复

- [ ] 根据答题结果更新 `reviewWeight`
- [ ] 实现按权重随机抽题

### Day 7：打磨与部署

- [ ] 完成响应式布局、空状态和可访问性检查

- [ ] 检查 API Key 方案是否满足部署安全要求

## v1.1：使用体验

- [ ] 导入 `.txt` 和 `.md` 文件
- [ ] 从 URL 抓取并清理正文
- [ ] 支持卡片搜索、筛选和批量删除
- [ ] 导出和导入 JSON，提供本地备份
- [ ] 支持 Anki 可导入格式

## v2：能力扩展

- [ ] 英文技术写作辅助：根据中文意图生成更自然的技术表达
- [ ] Prompt 版本选择与效果对比
- [ ] 记录失败样本，建立小型评估集
- [ ] 更完整的间隔重复算法和到期复习队列
- [ ] 在确认真实需求后评估账号、云同步和服务端代理

## 暂不考虑

- 日常场景词汇：已有成熟产品覆盖，不符合垂直定位。
- 社区、排行榜和社交功能：不能帮助验证核心学习闭环。
- 一开始就做多用户数据库：会显著增加认证、部署和隐私成本。
- 无限制长文档处理：成本和输出质量不可控，应先建立分段策略和限额。
