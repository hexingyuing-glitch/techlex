## 今天要做什么

- 封装 Claude API 请求，隔离鉴权、超时和错误映射。
- 将用户文本转换为 5-8 个术语及原文句子的结构化结果。
- 对模型输出做 JSON 解析、字段校验、去重和错误提示。

## 计划实现

`claudeApi.js` 只处理模型调用，不理解术语业务；`termExtractor.js` 负责注入 Prompt、解析 `{ terms: [...] }` 并返回干净数据。UI 需要保留原输入，在请求中禁用重复提交，并区分认证失败、限流、网络错误和格式错误。

不能把“要求 JSON”当作解析保障。模型响应进入组件状态前，必须确认 `terms` 是数组，每项的 `term` 与 `context_sentence` 都是非空字符串，数量合理，并对术语做大小写归一化去重。

## 预期问题与处理方式

### 问题 1：模型返回 Markdown 代码围栏

- **原因**：即使要求纯 JSON，模型也可能使用 ```json 包裹结果。
- **处理**：解析层可有限度移除最外层代码围栏；其他不合法内容应报错，避免使用脆弱正则猜测结构。

### 问题 2：提取出过于普通的词

- **原因**：模型容易把高频词误判为值得学习的术语。
- **处理**：Prompt 加入选择标准和反例，并建立固定样本记录低价值结果，后续按样本迭代。

### 问题 3：外部文本包含提示注入

- **原因**：用户粘贴的文档可能包含命令式文本。
- **处理**：使用明确的数据标签包裹原文，声明其中指令无效，并对返回结构做严格校验。

## 设计上的小思考

术语提取的核心不是“找到所有技术词”，而是做学习价值排序。宁可返回 4 个准确表达，也不应为了满足数量加入宽泛词汇。
